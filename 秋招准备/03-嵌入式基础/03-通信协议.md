## I2C — 面试问得最细的协议

### 物理层
- **两根线**：SCL（时钟）、SDA（数据）
- **必须上拉**：开漏输出，靠上拉电阻拉高（典型 4.7kΩ）
- **速度**：标准 100kHz / 快速 400kHz / 高速 3.4MHz

### 时序（必须能手画）
```
起始条件：SCL 高电平时，SDA 下降沿
停止条件：SCL 高电平时，SDA 上升沿

SDA 在 SCL 低电平时变化，在 SCL 高电平时被采样

     起始     地址+读写   应答    数据字节    应答     停止
SCL: ‾\_/‾‾/‾‾/‾‾/‾‾/‾‾/‾‾/‾‾/‾‾/‾‾/‾‾/‾‾/‾‾/‾‾/‾‾/‾
SDA: ‾\___/‾‾\__/‾\_/‾‾\__/‾\_/‾‾\__/‾\_/‾‾\__/‾\_/‾‾\__/‾‾‾/‾
      │  │    │  :  │    │  :  │    │  :  │    │  :     │
      └──┘    └──A6~A0┘ R/W┘ACK┘ └──D7~D0──┘ACK┘      停止
```

### 地址格式（7位地址）
- 7 位地址 + 1 位读写位 = 1 字节
- 读：`(addr << 1) | 1`
- 写：`(addr << 1) | 0`
- MPU6050 地址 0x68 → 写地址 0xD0，读地址 0xD1

### 应答机制
- 每字节后跟 1 个 ACK 位
- 接收方拉低 SDA = ACK（收到）
- 接收方不拉低 SDA = NACK（没收到/结束传输）

### 面试必问：为什么需要上拉电阻？
开漏输出只能拉低，不能主动拉高。上拉电阻把总线拉到高电平（空闲状态）。不用上拉，总线永远浮空。

### 面试必问：I2C 多机通信怎么仲裁？
- 每发一位就回读总线电平
- 如果自己发 1 但读到 0 → 说明有人同时发 0 → 自己失去仲裁，退出
- 谁先发 0 谁赢（因为开漏的线与特性：任一人拉低，总线就低）

## SPI — 高速全双工

### 物理层
- **四根线**：SCK（时钟）、MOSI（Master Out Slave In）、MISO（Master In Slave Out）、CS/SS（片选）
- **全双工**：同时收发
- **速度**：通常几 MHz 到几十 MHz
- **无标准协议层**：数据格式由设备定义

### 四种模式（CPOL + CPHA）
| 模式 | CPOL | CPHA | 时钟空闲电平 | 采样沿 |
|------|------|------|-------------|--------|
| 0 | 0 | 0 | 低 | 第1个沿（上升沿） |
| 1 | 0 | 1 | 低 | 第2个沿（下降沿） |
| 2 | 1 | 0 | 高 | 第1个沿（下降沿） |
| 3 | 1 | 1 | 高 | 第2个沿（上升沿） |

### 你 DS1302 的"模拟三线 SPI"
DS1302 不是标准 SPI，是类似 SPI 的三线协议：
- CE（使能，类似 CS）
- I/O（双向数据，类似 MOSI+MISO 合在一起）
- SCLK（时钟）
- 需要手动切换 I/O 方向（你在简历里写了"处理 I/O 方向切换"）

## UART — 异步串行

### 物理层
- **两根线**：TX（发）、RX（收）
- **异步**：无时钟线，双方约定波特率
- **帧格式**：起始位(1) + 数据位(5~9) + 校验位(可选) + 停止位(1/1.5/2)

### 波特率计算
```
波特率 = 总线时钟 / (16 × USARTDIV)

例：72MHz，目标 115200
USARTDIV = 72,000,000 / (16 × 115200) = 39.0625
整数部分=39，小数部分=0.0625 → DIV_Mantissa=39, DIV_Fraction=1
```

### 面试常见场景
- TTL UART：MCU 直连（3.3V/5V 电平）
- RS232：±15V 电平，需电平转换芯片（MAX232）
- RS485：差分信号，半双工，需方向控制（你的项目里可能没涉及，但面试可能问）

### 你的闹钟项目：UART 控制音乐芯片
- 单片机通过 UART 发送指令到音乐播放模块
- 格式类似 `AT+PLAY=1\r\n`，通过串口指令集控制

## 三协议对比（面试必背）

| | I2C | SPI | UART |
|---|-----|-----|------|
| 线数 | 2 | 3+N（N=从机数） | 2 |
| 通信方式 | 半双工 | 全双工 | 全双工 |
| 速度 | 100k-3.4M | 数M-数十M | 通常 ≤ 1M |
| 主从 | 多主多从 | 一主多从 | 点对点 |
| 地址 | 7/10位地址 | CS 片选 | 无 |
| 校验 | ACK 硬件确认 | 无（需上层协议） | 可选奇偶校验 |
| 距离 | 板级 | 板级 | 可达数十米 |
