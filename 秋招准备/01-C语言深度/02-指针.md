## 指针基础——一张图搞清楚

```
int a = 10;
int *p = &a;

内存视角：
地址:  0x20000100    0x20000104
       +--------+    +--------+
       |   10   |    |0x200.. |
       +--------+    +--------+
变量名:    a             p

p 的值 = &a（a的地址 = 0x20000100）
*p = a = 10
```

## 指针数组 vs 数组指针（面试高频混淆）

### 指针数组（array of pointers）
```c
int *arr[5];  // arr 是数组，有5个元素，每个元素是 int*
// 从右往左读：arr is an array [5] of pointers to int

// 典型用途：字符串数组（嵌入式菜单/命令表）
const char *menu[] = {"Start", "Stop", "Reset", "Config"};
// menu[0] 是 char*，指向 "Start"
```

### 数组指针（pointer to array）
```c
int (*ptr)[5];  // ptr 是指针，指向一个 int[5] 的数组
// 从内向外读：ptr is a pointer to an array [5] of int

int data[5] = {1, 2, 3, 4, 5};
int (*ptr_to_arr)[5] = &data;
// *ptr_to_arr 是整个数组
// (*ptr_to_arr)[2] = data[2] = 3
```

### 记忆口诀
- `int *arr[5]`：`[]` 优先级高于 `*`，所以 arr 先是数组，再是指针 → 指针数组
- `int (*ptr)[5]`：括号强制 `*` 先结合，所以 ptr 先是指针，再是数组 → 数组指针

### 面试题：二维数组参数传参
```c
void func(int arr[][4], int rows);     // 正确，第二维必须指定
void func(int (*arr)[4], int rows);    // 等价写法
void func(int **arr, int rows);        // 错误！二维数组不是二级指针
```

## 函数指针——嵌入式状态机的灵魂

### 基础用法
```c
// 定义函数指针类型
typedef void (*task_func_t)(void *arg);

// 使用
void led_task(void *arg) { /* LED闪烁 */ }
void sensor_task(void *arg) { /* 传感器采集 */ }

task_func_t func = led_task;  // 函数名本身就是地址
func(NULL);                   // 调用
```

### 嵌入式实战：状态机
```c
// 你闹钟项目的多模式切换，本质就是这个
typedef enum { MODE_CLOCK, MODE_ALARM, MODE_SET, MODE_MUSIC, ... } mode_t;
typedef void (*mode_handler_t)(uint8_t key_event);

mode_handler_t mode_table[6] = {
    clock_handler,
    alarm_handler,
    set_handler,
    ...
};

void main_loop(void) {
    mode_table[current_mode](key_event);  // 回调，解耦按键和模式
}
```

### 面试必问：回调函数机制
- 上层模块定义一个函数指针，下层模块在特定事件发生时通过这个指针调用上层函数
- 嵌入式应用：定时器回调、中断回调、FreeRTOS 定时器回调

## 野指针 / 空指针 / 悬空指针——三个坑

| 类型 | 定义 | 示例 |
|------|------|------|
| 野指针 | 未初始化的指针 | `int *p; *p = 10;` — 随机地址，危险 |
| 空指针 | 指向 NULL 的指针 | `int *p = NULL;` — 合法，但不能解引用 |
| 悬空指针 | 指向已释放内存的指针 | `free(p);` 后 p 还指向原地址 |

### 防护措施
```c
// 1. 声明时初始化
int *p = NULL;

// 2. free 后置 NULL
free(p);
p = NULL;

// 3. 使用前判空
if (p != NULL) {
    *p = 10;
}
```

## 指针与 const 的组合（再强化一遍）

```c
int a = 10, b = 20;

const int *p1 = &a;    // *p1 不可改，p1 可改
// p1 = &b; ✓
// *p1 = 30; ✗

int * const p2 = &a;   // p2 不可改，*p2 可改
// p2 = &b; ✗
// *p2 = 30; ✓

const int * const p3 = &a; // 都不能改
```

### 快速判断法
看 const 在 `*` 的哪边：
- const 在 `*` 左边 → 指向的内容只读
- const 在 `*` 右边 → 指针本身只读

## sizeof 与指针的陷阱

```c
void func(int arr[]) {
    // arr 已退化为指针
    printf("%d\n", sizeof(arr));  // 4（指针大小），不是数组大小！
}

int data[10];
printf("%d\n", sizeof(data));  // 40（在定义处是数组）
func(data);                    // 传入后退化
```

### 面试常考
```c
char *p = "hello";
sizeof(p);   // 4（指针大小）
strlen(p);   // 5（字符串长度）

char arr[] = "hello";
sizeof(arr); // 6（含 '\0'）
strlen(arr); // 5
```
