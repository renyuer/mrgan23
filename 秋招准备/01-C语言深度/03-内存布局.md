## 嵌入式程序的内存画像（必背）

```
高地址
+------------------+
|      栈 (Stack)   | ← SP（自动增长/收缩，局部变量、函数调用帧）
+------------------+
|        ↓          |
|                   |
|        ↑          |
+------------------+
|      堆 (Heap)    | ← malloc/free 管理
+------------------+
|   BSS 段          | ← 未初始化/初始化为0的全局/静态变量
+------------------+
|   DATA 段         | ← 已初始化（非0）的全局/静态变量
+------------------+
|   .rodata         | ← const 常量、字符串字面量（FLASH）
+------------------+
|   代码段 (.text)   | ← 程序指令（FLASH）
+------------------+
低地址
```

### 五段速查

| 段 | 存什么 | 谁管理 | 大小 |
|----|--------|--------|------|
| .text | 程序指令 | 编译期确定 | 固定 |
| .rodata | const 变量、字符串常量 | 编译期确定 | 固定 |
| DATA | 已初始化全局/静态变量 | 编译期确定 | 固定 |
| BSS | 未初始化全局/静态变量 | 编译期确定 | 固定 |
| 堆 | malloc 分配 | 程序员 | 动态 |
| 栈 | 局部变量、返回地址 | 编译器自动 | 动态 |

## 代码验证：变量存在哪

```c
#include <stdio.h>
#include <stdlib.h>

int g_init = 10;           // DATA 段
int g_uninit;              // BSS 段（自动初始化为0）
static int s_init = 20;    // DATA 段
const int c_val = 30;      // .rodata（FLASH）

int main(void) {
    static int local_static = 40;  // DATA 段
    int local = 50;                // 栈
    char *p = malloc(100);         // p 在栈，*p 在堆

    printf("g_init  addr: %p\n", &g_init);
    printf("g_uninit addr: %p\n", &g_uninit);
    printf("c_val   addr: %p\n", &c_val);
    printf("local_static addr: %p\n", &local_static);
    printf("local   addr: %p\n", &local);
    printf("p       addr: %p\n", p);
    // 观察地址谁在高位（栈）谁在低位（DATA/BSS）

    free(p);
    return 0;
}
```

## malloc / free 配对大忌

### 面试必考的内存泄漏场景
```c
// 场景1：忘记 free
void leak1(void) {
    char *buf = malloc(1024);
    // 函数返回，buf 指针消失，但 1024 字节的堆内存永远丢失
}

// 场景2：提前 return
void leak2(int err) {
    char *buf = malloc(1024);
    if (err) return;   // 泄漏！
    free(buf);
}

// 场景3：指针覆盖
char *p = malloc(100);
p = malloc(200);  // 第一次的 100 字节再也找不到了
free(p);
```

### 嵌入式最佳实践
```c
// 1. 分配后立即判空
char *buf = malloc(1024);
if (buf == NULL) { /* 错误处理 */ }

// 2. free 后置 NULL
free(buf);
buf = NULL;

// 3. 能用静态分配就不用动态分配（嵌入式铁律）
#define MAX_BUF 256
static char buf[MAX_BUF];  // 编译期确定，不存在泄漏
```

## sizeof 常见陷阱

### 数组参数退化
```c
void test(int arr[10]) {
    printf("sizeof(arr) = %d\n", sizeof(arr));  // 4 或 8（指针大小）
}
```

### 字符串
```c
char str1[] = "abc";   // sizeof = 4（含 '\0'），存在栈上
char *str2  = "abc";   // sizeof(str2) = 4/8（指针），字符串在 .rodata

// str1[0] = 'A'; ✓  可修改
// str2[0] = 'A'; ✗  未定义行为（.rodata 只读）
```

### 结构体（见下一章「结构体」）

### 动态数组不行
```c
int n = 10;
int arr[n];
sizeof(arr);  // 编译期 n 是变量，结果不确定（VLA 特殊处理）
```

## 栈溢出——嵌入式致命问题

### 原因
- 局部变量太多/太大
- 递归太深
- 中断嵌套过深（每个 ISR 压栈）
- FreeRTOS 任务栈太小

### 检测方法
```c
// 1. 栈填充检测（FreeRTOS 内置）
// 初始化时用 0xA5 填充任务栈
// 运行时检查栈底是否被覆盖

// 2. MPU 内存保护单元（硬件级）

// 3. 观察 HardFault_Handler
```

### 你的飞控项目
你在简历里写了「任务栈余量运行时验证」——面试时可以展开讲：
- 创建任务时分配栈空间
- FreeRTOS 的 `uxTaskGetStackHighWaterMark()` 返回最小剩余栈空间
- 或者手动填充 0xA5，运行时扫描被改写的区域来估算最大用量
