## 位运算六板斧

```c
#define BIT(n)      (1UL << (n))       // 第 n 位的掩码
#define SET_BIT(x,n)  ((x) |= BIT(n))   // 置位
#define CLR_BIT(x,n)  ((x) &= ~BIT(n))  // 清零
#define TGL_BIT(x,n)  ((x) ^= BIT(n))   // 翻转
#define GET_BIT(x,n)  (((x) >> (n)) & 1) // 读位
#define SET_BITS(x,mask,val) ((x) = ((x) & ~(mask)) | ((val) & (mask))) // 写多位
```

### 嵌入式必备用法
```c
// GPIO 操作（STM32 寄存器风格）
GPIOC->ODR |=  (1 << 13);   // PC13 置高（LED 亮）
GPIOC->ODR &= ~(1 << 13);   // PC13 置低（LED 灭）
GPIOC->ODR ^=  (1 << 13);   // PC13 翻转

// 判断按键按下（检测 PB0 是否为低）
if ((GPIOB->IDR & (1 << 0)) == 0) { ... }

// 配置寄存器：不改动其他位，只改目标位
#define MODE_MASK   0x03   // 两位模式位
#define MODE_OUTPUT 0x01
GPIOA->CRL = (GPIOA->CRL & ~MODE_MASK) | MODE_OUTPUT;
// 先清零目标位，再写入新值 —— 读-改-写
```

## 大小端（Endian）—— 面试必考

### 定义
- **大端（Big Endian）**：高字节在低地址
- **小端（Little Endian）**：低字节在低地址

```
uint32_t val = 0x12345678;
假设 val 在地址 0x20000000

大端：
0x20000000: 0x12
0x20000001: 0x34
0x20000002: 0x56
0x20000003: 0x78

小端（ARM Cortex-M 默认）：
0x20000000: 0x78
0x20000001: 0x56
0x20000002: 0x34
0x20000003: 0x12
```

### 判定代码
```c
int is_little_endian(void) {
    uint16_t val = 0x0001;
    return *(uint8_t *)&val;  // 小端返回1，大端返回0
}

// 或 union 法
int is_little_endian2(void) {
    union {
        uint16_t s;
        uint8_t  c;
    } u = {0x0001};
    return u.c;
}
```

### 大小端转换
```c
// 16位翻转
uint16_t swap16(uint16_t x) {
    return (x >> 8) | (x << 8);
}

// 32位翻转（你的飞控协议会用到）
uint32_t swap32(uint32_t x) {
    return ((x >> 24) & 0xFF) |
           ((x >>  8) & 0xFF00) |
           ((x <<  8) & 0xFF0000) |
           ((x << 24) & 0xFF000000);
}
```

### 嵌入式场景
- 你的飞控 2.4G 自定义协议：帧头+包序号+CRC 都在 uint8_t 缓冲区里，字节序由你控制，发收两端要一致
- I2C/SPI 发送多字节数据时要注意 MSB-first 还是 LSB-first
- 网络协议通常是「网络字节序=大端」，嵌入式设备（小端）要转换

## 统计二进制中 1 的个数（面试手写题）

```c
// 法1：逐位检测
int count_ones1(uint32_t n) {
    int cnt = 0;
    while (n) {
        if (n & 1) cnt++;
        n >>= 1;
    }
    return cnt;
}

// 法2：n & (n-1) 消除最低位1（更快）
int count_ones2(uint32_t n) {
    int cnt = 0;
    while (n) {
        n &= n - 1;  // 每次消掉最低位的1
        cnt++;
    }
    return cnt;
}
// 面试官问：为什么 n & (n-1) 能消掉最低位1？
// 答：n-1 把最低位1变成0，其右边所有0变成1，再 & 就清掉了

// 法3：查表法（嵌入式常用，时间换空间或反过来）
static const uint8_t bit_count[256] = {0,1,1,2,1,2,2,3, ...};
int count_ones3(uint32_t n) {
    return bit_count[n & 0xFF] +
           bit_count[(n>>8) & 0xFF] +
           bit_count[(n>>16) & 0xFF] +
           bit_count[(n>>24) & 0xFF];
}
```

## 位段（bit-field）

```c
// 协议解析常用
struct packet_header {
    uint16_t seq   : 8;   // 包序号，占8位
    uint16_t type  : 4;   // 包类型，占4位
    uint16_t ack   : 1;   // 确认位，占1位
    uint16_t rsvd  : 3;   // 保留，占3位
};
// 总共正好 16 位
```

### 注意
- 位段的存储顺序和字节对齐是编译器相关的，跨平台通信不推荐用位段解析协议
- 在 MCU 内部做标志位压缩存储可以用
