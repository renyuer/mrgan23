## 三段式详述（面试 2-3 分钟版）

### 1. 做什么
为深入学习嵌入式实时控制和传感器融合，从零设计完整的飞控 + 遥控系统。飞控端负责姿态解算、PID 控制和电机驱动；遥控端负责摇杆采集和数据收发。两端通过 2.4G 模块双向通信。

### 2. 关键挑战 & 解决

**挑战1：FreeRTOS 实时任务划分**
- 将系统划分为 5 个任务：姿态解算(最高优先级/1kHz)、飞控(1kHz)、通信(250Hz)、OLED显示(20Hz)、状态监控(10Hz)
- `vTaskDelayUntil` 保证控制周期绝对精确，不随时间漂移 ← **面试这句话必说**
- 任务间通过队列传递传感器数据和指令，互斥锁保护全局状态
- 开发阶段用 `uxTaskGetStackHighWaterMark()` 验证每个任务的栈余量，压测后确认分配合理

**挑战2：MPU6050 驱动与姿态解算（自写，未用 DMP）**
- 独立编写 I2C 驱动读取 MPU6050 原始数据（加速度计 + 陀螺仪）
- **为什么不用 DMP**：DMP 是黑盒，我想深入理解姿态解算原理；自己写的代码可控、可调试、可优化
- 四元数互补滤波做姿态解算：
  - 加速度计提供长期稳定的角度参考（通过重力方向反算倾斜角）
  - 陀螺仪提供短期快速的角速度积分
  - 互补滤波融合两者：高频信陀螺仪，低频信加速度计
- 低通滤波抑制电机振动带来的高频噪声
- 静态校准：上电后采集 500 组静止数据做零偏校正
- **IMU 异常检测**：全零检测 → 传感器未初始化；数据冻结检测 → 通信中断；数值溢出 → 异常数据 → 触发紧急保护，缓慢降低油门

**挑战3：串级 PID 控制**
- 外环：角度环（仅 P）——根据期望角度和当前角度的偏差，给出目标角速度
- 内环：角速度环（PD）——根据目标角速度和陀螺仪实测角速度的偏差，输出电机控制量
- **为什么串级而不是单环**：单环 PID（角度 → 电机）响应慢，加入内环角速度后能更快抑制扰动，悬停更稳
- 调试策略：先调内环 PD 到角速度跟踪收敛且无振荡，再接入外环 P 整定
- 积分限幅防饱和：限制积分项不超过电机输出范围的 30%
- 分轴控制：Roll/Pitch/Yaw 三轴独立 PID

**挑战4：自定义 2.4G 通信协议**
- 帧结构：帧头(1B 0xAA) + 包序号(1B) + 数据(可变长度) + CRC32(4B)
- 250Hz 双向通信，接收端做丢包检测（连续序号不连续 → 丢包）
- 连续丢包超过阈值 → 自动锁定（飞控端紧急降落，遥控端报警）
- **为什么 CRC32 而不是 CRC8**：2.4G 环境干扰大，CRC32 能检测更多位错误

**挑战5：资源限制下的优化**
- 代码 < 32KB Flash：编译优化 -Os，只链接用到的库函数
- RAM < 12KB：任务栈大小逐一测算（姿态解算 512B、飞控 384B、通信 256B、OLED 256B、监控 128B），队列使用指针引用而非拷贝减少内存占用

### 3. 成果
室内无风环境稳定悬停，姿态跟踪平稳。代码在 STM32F103C8T6 上运行，Flash 和 RAM 都在芯片限制内。

## 预判追问（面试官可能怎么追问，你怎么答）

### 姿态解算
**Q: 互补滤波和卡尔曼滤波有什么区别？为什么不选卡尔曼？**
A: 互补滤波计算量小，只需调整一个互补系数 α；卡尔曼滤波更精确但计算量大、需要已知噪声协方差矩阵。微型四轴计算资源有限（STM32F103 72MHz），互补滤波足够了。而且我实测互补滤波就能达到稳定悬停的效果。

**Q: 互补系数怎么选的？**
A: 通常 α 取 0.96~0.98（陀螺仪权重）。我实测从 0.90~0.99 梯度测试，0.96 时姿态平滑且对振动不敏感。α 越大越信陀螺仪（响应快但温漂累积快），α 越小越信加速度计（响应慢但稳定）。

**Q: 加速度计怎么反算姿态角？**
A: 重力加速度 [0, 0, -1g]，测量到的三轴分量通过反正切得出 roll 和 pitch：`roll = atan2(ay, az)`，`pitch = atan2(-ax, sqrt(ay²+az²))`。但 yaw 不能直接从加速度计得到（绕重力轴旋转加速度不变），需要陀螺仪积分。

### PID 相关
**Q: 串级 PID 比单环 PID 好在哪？**
A: 单环 PID 只根据角度误差输出，响应慢——飞机歪了才反应。内环角速度环先感知角速度变化，在角度偏差还小的时候就产生阻尼，响应速度更快、抗扰动更强。本质是多了一个"预测"层。

**Q: 积分限幅怎么设的？**
A: 积分项限制在 ±30% 输出范围。超过这个值说明有持续偏差但 PID 参数不够纠正，积分继续累积只会导致饱和（积分饱和会让飞机突然"冲"一下）。

**Q: PID 参数怎么调的？**
A: 经验法 + 试凑。先定 KP（给一个阶跃期望角度，观察超调和振荡），调大到微微振荡 → 退回 60%。然后加 KD 抑制振荡，最后小量 KP 微调。

### FreeRTOS 相关
**Q: 为什么飞控任务用 1kHz？这个频率怎么定的？**
A: 四轴的机械响应带宽大约在 100-200Hz，控制频率取 5-10 倍余量 → 1kHz。太慢控制不平滑，太快 CPU 吃不消。实测 1kHz 在 STM32F103 上姿态解算+控制约 400μs，还有余量。

**Q: 队列传数据你说用引用方式，怎么保证发送方不提前覆盖？**
A: 发送方 `xQueueSend` 后不能立即修改该内存，因为接收方可能还没处理完。我的做法是双缓冲——两个 buffer 交替使用，发送完一个后切换到另一个。

### 通信协议相关
**Q: 250Hz 通信频率怎么选的？**
A: 控制频率 1kHz，通信每 4 个控制周期发一次（4 × 1ms = 4ms → 250Hz），在延迟和带宽之间折中。

**Q: 丢包超时自动锁定怎么实现？**
A: 维护一个连续丢包计数器。收到正确包（序号连续递增）→ 清零计数器；序号不连续 → 计数器累加。计数器超过阈值（比如 50 包 ≈ 200ms）→ 触发锁定：飞控端油门缓慢归零，遥控端 OLED 提示"连接丢失"。

### IMU 异常检测
**Q: 怎么检测传感器数据"冻结"？**
A: 每次读取后比较本次和上次的数据。连续 N 次（比如 10 次）完全相同 → 判断为冻结。正常情况下即使静止，传感器噪声也会让 LSB 跳动。
